372/* SWIM FISHY SWIM TILEMAP FORMAT
373 MAP DATA IS STORED AS BIG ENDIAN AUTOMATICALLY -> Although endian support will be incorporated
374
3751. TilemapSize Data:
376 short tilemapWidth; (2 bytes) - Describes the width of the actual tilemap in tiles
377 short tilemapHeight; (2 bytes) - Describes the height of the actual tilemap in tiles
378 short graphicWidth; (2 bytes) - Descirbes the width of the graphic file in pixels
379 short graphicHeight; (2 bytes) - Describes the height of the graphic file in pixels
380
3812. TilemapData Layer A:
382 int mapIndexDataX[tilemapWidth * tilemapHeight]; - Describes the X graphic index for each tile in the tilemap
383 int mapIndexDataY[tilemapWidth * tilemapHeight]; - Describes the Y graphic index for each tile in the tilemap
384 (The above two arrays should be loaded into an vector of TextureIndex structs)
385
3863. TilemapData Layer B:
387 int mapIndexDataX[tilemapWidth * tilemapHeight]; - Describes the X graphic index for each tile in the tilemap
388 int mapIndexDataY[tilemapWidth * tilemapHeight]; - Describes the Y graphic index for each tile in the tilemap
389 (The above two arrays should be loaded into an vector of TextureIndex structs)
390
3914. TilemapData Layer C:
392 int mapIndexDataX[tilemapWidth * tilemapHeight]; - Describes the X graphic index for each tile in the tilemap
393 int mapIndexDataY[tilemapWidth * tilemapHeight]; - Describes the Y graphic index for each tile in the tilemap
394 (The above two arrays should be loaded into an vector of TextureIndex structs)
395
3965. TilemapData Layer D:
397 int mapIndexDataX[tilemapWidth * tilemapHeight]; - Describes the X graphic index for each tile in the tilemap
398 int mapIndexDataY[tilemapWidth * tilemapHeight]; - Describes the Y graphic index for each tile in the tilemap
399 (The above two arrays should be loaded into an vector of TextureIndex structs)
400
4015. Collision Layer:
402 bool mapCollisionData[tilemapWidth * tilemapHeight]; - Describes collisions by (x * y) indexing. true if collision on tile, false if not.
403
4046. Z Layer:
405 QuadZValues mapZData[tilemapWidth * tilemapHeight]; - Describes z values as a set of 4 bytes for each tile... I know it should technically be for each corner... but this let's us get away with some normal looking tilemap combinations...
406
4077. That's the entire file!
408
409*/
